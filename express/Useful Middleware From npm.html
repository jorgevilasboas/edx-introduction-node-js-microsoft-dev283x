Middleware pattern is a series of processing units connected together, where the output of one unit is the input for the next one. In Node.js, this often means a series of functions in the form:</p> <pre><code class="language-js">function(args, next) { // ... Run some code next(output) // Error or real output } </code></pre> <p>The middleware pattern implements continuity. The request is coming from a client and a response is sent back to the client.</p> <pre><code>request-&gt;middleware1-&gt;middleware2-&gt;...middlewareN-&gt;route-&gt;response </code></pre> <p>Express has a middleware manager so developers only need to apply the middleware with <code>app.use()</code>. Take a look at this example:</p> <pre><code class="language-js">var express = require('express') var app = express() //... Define middleware1-N app.use(middleware1) app.use(middleware2) ... app.use(middlewareN) ... </code></pre> <h3>Middleware Order</h3> <p>Middleware are executed in the order specified. In the following example, the <code>logger('dev')</code> middleware causes requests to be logged before the <code>bodyParser.json()</code> middleware causes payloads of requests to be parsed into <code>request.body</code>.</p> <pre><code class="language-js">var logger = require('morgan') var bodyParser = require('body-parser') ... app.use(logger('dev')) app.use(bodyParser.json()) </code></pre> <h3>Two Types of Express Middleware</h3> <p>There are two types of Express middleware:</p> <ol> <li>npm modules, e.g., <code>body-parser</code> from npm used with <code>app.use(bodyParser.json())</code></li> <li>Custom middleware, e.g., <code>app.use((req, res, next)=&gt;{next()})</code></li> </ol> <p>Developers can mix them. By utilizing middleware, developers can modularize their applications and reuse code.</p> </div> </div> </div> </div>

Here's an example of creating a middleware and saving it as a variable which is used later to apply the middleware to the Express app:</p> <pre><code class="language-js">const middleware = (request, response, next) =&gt; { // Modify request or response // Execute the callback when done next() } app.use(middleware) </code></pre> <p>Developers can also create middleware right in the <code>app.use()</code> call using an anonymous function definition:</p> <pre><code class="language-js">app.use((request, response, next) =&gt; { // Modify request or response // Execute the callback when done next() }) </code></pre> <p>The first approach is better for modularizing because it's easier to move a named function or a function saved into a variable than an anonymous function.</p> <h3>Passing References</h3> <p><code>request</code> is <strong>always</strong> the same object in the lifecycle of a single client request to the Express server. This allows us to implement a very useful pattern in which developers pass data from one middleware to another and to a request handler.</p> <p>For example, developers can connect to a database in one middleware and make it available in all the subsequent middleware functions and request handlers (routes).</p> <pre><code class="language-js">app.use(function (request, response, next) { DatabaseClient.connect(URI, (err, db) =&gt; { // error handling request.database = db next() }) }) </code></pre> <p>In this middleware, <code>database</code> is available in the request object and we can run queries such as finding an application by the app ID:</p> <pre><code class="language-js">app.use((req, res, next) =&gt; { req.database.collection('apps').findOne({appId: req.query.appId}, (err, app) =&gt; { // error handling req.app = app next() }) }) </code></pre> <p>This makes moving routes and middleware to other files (modularization) straightforward, i.e., keeping code neat and organized.</p> </div> </div> </div> </div>

request body, a.k.a. payload, has information from a POST request when a client wants to create a new entity/record or update an existing one with PUT. Developers who implement servers, often need to access the request body information.</p> <p>There's body-parser npm module which does just that. No need to code anything. Just install <code>body-parser</code> and enable the <code>json()</code> and <code>urlencoded()</code> middleware to convert raw form data into JSON.</p> <p>So first, install body-parser with npm:</p> <pre><code>npm install body-parser </code></pre> <p>Then import middleware:</p> <pre><code class="language-js">const bodyParser = require('body-parser') </code></pre> <p>And apply <code>json</code> to parse <code>application/json</code> type (that's what single-page applications and other JSON REST clients use):</p> <pre><code class="language-js">app.use(bodyParser.json()) </code></pre> <p>Usage: AJAX/XHR from single-page applications and other JSON REST clients.</p> <p>Sometimes, the type could be different from <code>application/json</code>. For example, HTML web forms <code>&lt;form&gt;</code> with <code>action</code> attribute use <code>application/x-www-form-urlencoded</code>. Useful Middleware From npm

There are a lot of middleware modules to use. Every one of them is installed with the npm command using the module's npm name:

npm install <package_name> --exact

Here's the most popular and useful Express middleware which you'll probably end up using in almost every project (listed the npm names):

  • body-parser request payload
  • compression gzip
  • connect-timeout set request timeout
  • cookie-parser Cookies
  • cookie-session Session via Cookies store
  • csurf CSRF
  • errorhandler error handler
  • express-session session via in-memory or other store
  • method-override HTTP method override
  • morgan server logs
  • response-time
  • serve-favicon favicon
  • serve-index
  • serve-static static content
  • vhost: virtual host
  • cookies and keygrip: analogous to cookieParser
  • raw-body: raw request body
  • connect-multiparty, connect-busboy: file upload
  • qs: analogous to query
  • st, connect-static analogous to staticCache
  • express-validator: validation
  • less: LESS CSS
  • passport: authentication library
  • helmet: security headers
  • connect-cors: CORS
  • connect-redis

Express is a minimalistic framework. By using only the middleware you need for a particular project, you end up with a custom framework which is not bulky and doesn't have anything extra which your project doesn't need or use. The vast number of open-source middleware modules allows you to pick and choose and then modify the behavior of your application thanks to the flexible and configurable design of Express.

Feel free to print this list of commonly used Express middleware or download cheatsheet from GitHub.
